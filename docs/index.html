M1 → M7 · 标准化 CFD 工作流

不是"想哪写哪"。NullFlux 把 OpenFOAM 任务拆成 7 个阶段，每阶段必须通过 *_verify 视觉核验才能进下一步。

M1 · GEOM

几何与单位

STL 检查、单位核对、法向 / 封闭性 VLM 体检。

M2 · PLAN

方案与参数

论文参数提取 + 三遍核对 + 锁定到 paper_params.lock.md。

M3 · MESH

网格生成

blockMesh + snappyHexMesh，自动 STL 贴合度采样。

M4 · BC

边界与 0.orig

必走 0.orig/ 工作流，改飞一行回滚。

M4.5 · IC

初始场 setFields

多相 / 分层算例的"算白算"防御。

M5 · SOLVE

求解器异步运行

残差时序流式抓取，发散自动停。

M6 · POST

后处理与对比

ParaView 离屏渲染 → VLM 体检 → 与论文图对比。

M7 · OPT

参数寻优

贝叶斯优化 study，baseline 通过后才允许开。

系统要求

  • Node.js · ≥ 18
  • Python · ≥ 3.9（含 numpy / PyMuPDF）
  • OpenFOAM · v8/v10/v2306（可选，跑 FOAM 任务必备）

常见问题

先看这里，省得 issue 里互相浪费时间。

NullFlux 和 ChatGPT / Cursor 有什么区别？
通用编程助手不懂 CFD 流程：它不知道 blockMesh 失败 80% 是 STL 单位错、不知道 0.orig/ 工作流、不会盯着残差曲线判断震荡型发散。NullFlux 内置全套 CFD 工程协议 + 视觉核验 + 论文复现专用工作流，并且能直接驱动本地的 OpenFOAM / MFIX / ParaView。
支持哪些大模型？
默认接 SiliconFlow / DeepSeek / Kimi（VLM）/ Moonshot / 通义 / GLM / 智谱 / OpenAI / Claude / Gemini。也支持本地 vLLM / Ollama / LM Studio（OpenAI 兼容协议）。视觉核验默认用 Kimi-VL 或 Qwen-VL。
开源吗？为什么是封装混淆发行？
当前是 封装发行版（sealed）：esbuild 打包 + javascript-obfuscator（RC4 + 控制流扁平化）。源码暂未公开，欢迎付费使用 / 商务合作。封装版完全可用、可本地运行、可二次配置。
数据安全吗？我的算例会被上传吗？
case 文件、几何、结果、token 默认全部留在本机文件系统。NullFlux 本身不连任何上传服务器。**唯一外发**：你选的模型 API（DeepSeek / OpenAI / …）会收到对话上下文。要完全离线，接本地 vLLM 即可。
Windows 能跑吗？
前端 UI 跑 Windows 没问题，但 OpenFOAM / pvpython 需要 Linux。**推荐方案**：Windows + WSL2（Ubuntu 22.04），NullFlux 跑在 WSL 里，浏览器从 Windows 访问。
遇到 Bug / 想提需求？
去 GitHub Issues 提，附上 server 终端日志 + case 路径。或者直接联系作者（README 末尾有联系方式）。
